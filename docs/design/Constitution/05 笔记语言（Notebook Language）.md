# BioassayCraft 产品设计宪章

# 第五章｜Notebook Language（笔记语言）

> 我们不是在设计网页。
>
> 我们是在设计一本能够思考的实验笔记。

---

# 为什么是 Notebook

Notebook 并不是一种视觉风格。

它是一种科学工作的方式。

对于研究人员来说，

真正重要的知识，

很少来自最终发表的论文。

更多来自：

实验记录。

推导过程。

旁边写下的一句话。

一次失败后的修改。

一次重新设计。

这些内容，

共同组成了科学真正发生的地方。

BioassayCraft 希望把这种体验带到数字世界。

---

# Notebook 不是背景

很多产品会使用：

网格纸。

坐标纸。

纸张纹理。

来表达 Notebook。

这只是视觉元素。

不是 Notebook。

真正的 Notebook，

应该让用户感觉：

这里正在发生思考。

因此，

Notebook 永远是一种表达方式，

而不是一种背景样式。

---

# 页面应该像正在写，而不是已经完成

传统网页，

给人的感觉是：

已经完成。

Notebook，

应该给人的感觉是：

正在形成。

因此，

页面中的内容，

不应该一次全部出现。

而应该随着理解逐渐形成。

例如：

一条线慢慢连接。

一个节点逐渐出现。

一句批注随后补充。

整个页面，

应该让用户感觉：

自己正在参与一次思考。

而不是阅读一次总结。

---

# Margin Notes

Notebook 最重要的元素，

不是正文。

而是页边批注（Margin Notes）。

优秀的导师，

不会在正文里解释所有事情。

他们会在旁边写下一句话。

例如：

"这里真正的问题不是精密度。"

或者：

"重新思考 ATP。"

又或者：

"为什么不是另一种模型？"

这些内容，

应该像实验笔记里的即时思考。

因此，

Margin Notes 应成为 BioassayCraft 的核心设计语言。

---

# 一句话胜过一段解释

BioassayCraft 不鼓励长篇说明。

真正重要的内容，

应该尽可能浓缩。

例如：

不要写：

"Acceptance Criteria should always be established according to ATP before validation."

更好的表达是：

"ATP first."

或者：

"Criteria follow purpose."

短句，

更容易成为思考锚点。

---

# 留白属于内容

很多设计把留白理解为：

没有内容。

BioassayCraft 认为：

留白本身就是内容。

留白意味着：

等待。

思考。

观察。

连接。

因此，

不要害怕页面很空。

真正优秀的 Notebook，

从来不是写满每一个角落。

---

# 不要一次解释所有事情

Notebook 的特点，

不是完整。

而是连续。

因此，

每一个页面，

都应该留下继续思考的空间。

不要试图一次解释：

所有概念。

所有法规。

所有统计。

用户应该始终感觉：

下一页还有新的理解。

---

# 保留推理过程

很多科学网站，

只展示最终结果。

Notebook 应展示：

推导过程。

例如：

模型为什么改变？

为什么放弃这个方案？

为什么重新设计实验？

这些内容，

比最终结果更重要。

用户应该能够看见：

思考如何发生。

---

# 可以允许不确定性

真正的实验笔记，

很少完全正确。

它允许：

修改。

划掉。

重新连接。

增加批注。

BioassayCraft 也应该允许这种表达。

例如：

一个节点可以重新连接。

一条路径可以改变。

一个假设可以被否定。

这些都不是错误。

而是科学的一部分。

---

# 不追求整齐

Notebook 的目标，

不是整齐。

而是真实。

因此，

页面不应过度追求：

绝对对齐。

绝对规则。

绝对对称。

适度的不规则，

能够体现思考过程。

但这种不规则，

必须是克制的。

它不能降低阅读效率。

---

# 每一个元素都应该像写下来的

所有视觉元素，

都应该回答：

它为什么会出现在实验笔记里？

例如：

节点。

箭头。

圈注。

页边批注。

简短总结。

这些都属于 Notebook。

而：

大型卡片。

营销 Banner。

Feature Grid。

Dashboard Panel。

这些不属于 Notebook。

---

# Notebook 的节奏

Notebook 不应该快速浏览。

它应该：

停顿。

思考。

继续。

因此，

滚动节奏应该缓慢。

动画应该克制。

解释应该简洁。

页面应该鼓励：

反复阅读。

而不是快速浏览。

---

# Notebook 的语气

Notebook 永远不是命令。

也不是宣传。

更不是教程。

它更像：

一位经验丰富的导师，

在实验记录旁边，

留下来的思考。

因此，

整个网站应避免：

"Click here"

"Best Practice"

"Powerful"

"Advanced"

这类营销语言。

更适合：

"Think again."

"Why?"

"Evidence?"

"Assumption?"

这些能够引发思考的语言。

---

# Notebook 与 Journey

Notebook 负责：

记录思考。

Journey 负责：

组织思考。

Notebook 更关注：

局部。

Journey 更关注：

整体。

两者共同组成：

BioassayCraft 的知识表达方式。

任何 Journey，

最终都应能够拆解成：

一页页 Notebook。

任何 Notebook，

最终都应能够连接回：

Journey。

---

# Notebook 的终极目标

当用户关闭网页之后，

真正留在脑海里的，

不应该是：

动画。

按钮。

颜色。

而应该是：

那些写在页边的一句话。

一个重新思考的问题。

一个曾经忽略的假设。

一个突然理解的概念。

如果用户愿意在自己的实验记录里，

写下同样一句批注。

那么，

BioassayCraft 就真正完成了 Notebook 的使命。

---

# 本章总结

Notebook，

不是一种设计风格。

也不是一种排版方式。

它是一种科学表达语言。

它允许思考被记录。

允许假设被修改。

允许问题继续存在。

允许知识逐渐形成。

BioassayCraft 的每一个页面，

都应让用户感觉：

自己正在阅读一本仍在不断完善的实验笔记。

因为真正的科学，

从来不是最终答案。

而是不断接近答案的过程。
