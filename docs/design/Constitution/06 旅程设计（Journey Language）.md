# BioassayCraft 产品设计宪章

# 第六章｜Journey Language（Journey 设计语言）

> Journey 不是流程。
>
> Journey 是科学推理留下来的轨迹。

---

# 为什么需要 Journey

科学工作并不是由一系列孤立的知识组成。

真正的实验，

并不会因为阅读了一章药典，

或者完成了一次计算，

就自动结束。

实验的发展，

来自一个又一个连续的科学决策。

因此，

BioassayCraft 不希望把知识组织成：

章节、

菜单、

模块、

或者功能。

我们希望组织的是：

推理如何不断向前发展。

Journey 正是这种发展的可视化。

---

# Journey 不是导航

很多网站，

Journey 等同于 Navigation。

例如：

首页

↓

产品

↓

功能

↓

联系我们

这不是 BioassayCraft 的 Journey。

Journey 从来不是告诉用户：

"下一页在哪里。"

Journey 告诉用户：

"下一步应该思考什么。"

因此，

Journey 服务于理解。

而不是服务于浏览。

---

# Journey 是推理，而不是流程

流程强调：

执行顺序。

Journey 强调：

思维演化。

例如：

Method Development

并不是：

第一步。

它只是某一个问题发展到一定阶段之后，

自然形成的新问题。

因此，

Journey 永远不是：

Step 1

↓

Step 2

↓

Step 3

Journey 应该表现为：

Question

↓

Reasoning

↓

Decision

↓

New Question

这是一个不断循环的过程。

---

# Journey 从 Question 开始

任何 Journey，

都必须从一个真实的问题开始。

而不是：

一个模块。

例如：

不要：

Method Validation

更好的开始是：

"Can this assay support its intended purpose?"

不要：

ANOVA

更好的开始是：

"Does this model explain the data well enough?"

不要：

Sample Size

更好的开始是：

"How much evidence is enough?"

用户应该进入问题。

而不是进入章节。

---

# Journey 不追求最短路径

现实中的科学，

几乎没有最短路径。

实验会失败。

模型会改变。

假设会被推翻。

设计会重新开始。

因此，

Journey 应该允许：

回头。

修改。

重新连接。

重新推导。

如果 Journey 永远只有一条直线，

它就失去了真实性。

---

# Journey 是不断展开的

Journey 不应该在第一页全部展示。

它应该随着理解逐渐形成。

用户第一次进入页面时，

只能看到：

当前问题。

随着滚动、

观察、

交互，

新的节点逐渐出现。

新的连接逐渐形成。

Journey 是成长出来的。

不是一次绘制完成的。

---

# Journey 永远保留上下文

新的节点出现时，

旧节点不应该完全消失。

因为：

科学推理，

从来不会忘记之前的假设。

例如：

ATP 出现之后，

即使进入 Validation，

ATP 仍应作为背景存在。

它提醒用户：

所有后续决策，

都来源于最初的问题。

因此，

Journey 应始终保留推理上下文。

---

# Journey 的节点

Journey 中，

节点不是页面。

也不是功能。

节点代表：

一个科学决策。

例如：

ATP

Model Selection

Acceptance Criteria

Validation Strategy

这些都不是知识点。

它们都是：

Decision Point。

每一个节点，

都必须回答：

这里正在决定什么？

---

# Journey 的连接

连接，

比节点更重要。

节点告诉用户：

发生了什么。

连接告诉用户：

为什么会发生。

因此，

线条不只是视觉元素。

它代表：

Reasoning。

每一条连接，

都应能够解释：

为什么这一节点会导向下一节点。

如果无法解释，

这条连接就不应该存在。

---

# Journey 应允许分支

现实中的 Bioassay，

很少只有一种正确方案。

例如：

同一个 ATP，

可能导向：

不同模型。

不同 Readout。

不同实验设计。

Journey 应允许这种分支。

因为：

真正重要的是：

为什么选择这一条路径。

而不是：

只有这一条路径。

---

# Journey 应允许汇聚

不同实验设计，

最终可能得到：

相同结论。

不同模型，

最终可能支持：

相同 Validation。

因此，

Journey 不仅能够分叉。

也能够重新汇聚。

这体现的是：

科学中的收敛。

---

# Journey 与 Tool

Tool 不属于 Journey。

Tool 长在 Journey 上。

例如：

Model Selection

↓

ANOVA Explorer

Acceptance Criteria

↓

Validation Sample Size Calculator

Tool 永远依附于：

Decision Point。

而不是：

Navigation。

---

# Journey 与 Learn

Learn 不应独立存在。

任何 Learn，

都来自 Journey。

例如：

用户遇到：

Residual。

这时：

Learn：

解释 Residual。

用户遇到：

Bias。

这时：

Learn：

解释 Bias。

学习，

应发生在真正需要的时候。

而不是：

提前灌输。

---

# Journey 的节奏

Journey 应保持：

停顿。

观察。

理解。

继续。

每一个节点，

都应该让用户完成一次思考。

之后，

Journey 才继续生长。

如果节点不断快速出现，

Journey 就会重新变成：

流程图。

---

# Journey 的终点

Journey 没有真正的终点。

Validation，

不是结束。

Routine，

也不是结束。

新的产品。

新的问题。

新的法规。

新的模型。

都会重新开始新的 Journey。

因此，

Journey 应始终保持：

开放。

它永远可以继续延伸。

---

# Journey 的设计原则

任何 Journey，

都应满足以下原则：

第一，

从问题开始。

第二，

展示推理，而不是流程。

第三，

允许分支。

第四，

允许回退。

第五，

保留上下文。

第六，

逐渐形成。

第七，

连接真实实践。

如果其中任何一项缺失，

Journey 就不完整。

---

# 本章总结

Journey，

不是网站导航。

不是产品结构。

不是课程目录。

Journey 是科学推理的可视化。

它记录：

问题如何出现。

假设如何形成。

证据如何积累。

决策如何发生。

新的问题又如何产生。

BioassayCraft 希望用户记住的，

不是 Journey 的路线。

而是：

一种不断提出问题、

不断修正理解、

不断建立证据、

不断完成科学决策的思考方式。

真正优秀的 Journey，

最终会消失。

留下来的，

只有用户自己的 Scientific Reasoning。
