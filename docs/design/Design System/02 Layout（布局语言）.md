# BioassayCraft Design System

# 第二章｜Layout（布局语言）

> 布局不是排列内容。
>
> 布局是在组织思考。

---

# 为什么先设计 Layout

大多数网页设计，

从颜色开始。

从字体开始。

从按钮开始。

BioassayCraft 不这样设计。

我们首先设计：

思考如何流动。

而 Layout，

正是这种思考流动的可视化。

页面不是：

放东西。

而是：

组织认知。

---

# Layout 服务于 Cognitive Flow

任何布局，

都必须回答：

用户此刻应该思考什么？

而不是：

还有哪里可以放内容？

因此，

Layout 的职责不是：

利用空间。

而是：

控制注意力。

---

# 一屏，一个思想

BioassayCraft 最重要的布局原则：

一个 Screen，

一个核心思想。

不要设计：

一个屏幕，

四张卡片。

六个按钮。

三个栏目。

一个 Hero。

一个 Sidebar。

这些都会让用户：

同时思考很多事情。

真正优秀的 Layout，

应该让用户：

一次只思考一件事情。

---

# 不要使用 Dashboard Layout

Dashboard：

适合管理系统。

BioassayCraft：

不是管理系统。

因此，

禁止：

Card Grid

Metric Panel

Sidebar Navigation

Dense Toolbar

Floating Widget

这些布局，

都会让用户产生：

浏览。

而不是：

思考。

---

# 不要设计 Feature Grid

现代 Landing Page 最喜欢：

四宫格。

六宫格。

Feature Card。

Icon Grid。

BioassayCraft 永远不要这样。

Feature Grid 会告诉用户：

这里有很多东西。

Scientific Narrative 应该告诉用户：

这里有一个问题。

---

# 布局必须具有方向

页面必须能够回答：

理解正在朝哪里发展？

因此，

Layout 必须具有：

Direction。

例如：

Question

↓

Reasoning

↓

Evidence

↓

Decision

不要设计：

随机排列。

随机漂浮。

随机分布。

用户必须能够感觉：

自己正在不断前进。

---

# 留白决定节奏

页面真正决定阅读速度的，

不是文字。

而是：

留白。

留白越少，

阅读越快。

留白越大，

思考越慢。

BioassayCraft 应主动利用留白，

控制阅读节奏。

不要害怕：

大片空白。

---

# Sticky 是为了理解

允许：

Sticky Layout。

但：

Sticky 不应该为了：

炫技。

而应该为了：

保持上下文。

例如：

ATP 保持。

Journey 生长。

Evidence 增加。

这样，

用户不会失去：

思考上下文。

---

# 上下文永远存在

新的内容出现，

旧内容不要立即消失。

因为：

科学推理，

始终依赖之前的假设。

例如：

Question

始终保留。

ATP

始终存在。

Journey

不断生长。

上下文，

比动画更重要。

---

# 左右布局的职责

推荐：

左侧：

Reasoning。

右侧：

Visualization。

或者：

左侧：

Question。

右侧：

Evidence。

不要：

左右完全相同的重要性。

页面必须存在：

主次关系。

---

# 不要追求对称

优秀网页，

未必对称。

优秀网页，

一定平衡。

BioassayCraft 不追求：

左右镜像。

上下镜像。

而追求：

视觉重心稳定。

---

# 页面应该拥有呼吸

不要：

内容填满整个屏幕。

任何页面，

至少应保留：

30~50%

的留白空间。

如果页面一直很满，

用户很快就会疲劳。

---

# 内容出现的顺序

页面中，

任何元素，

都必须回答：

为什么现在出现？

而不是：

为什么放在这里？

例如：

Question

↓

ATP

↓

Journey

↓

Evidence

↓

Tool

这是：

理解顺序。

不是：

布局顺序。

---

# 永远不要为了整齐而整齐

真正的 Notebook，

不会绝对整齐。

适度的不规则，

能够体现：

思考。

但是：

这种不规则，

必须具有逻辑。

不能降低阅读效率。

---

# Layout Checklist

每一个页面，

发布之前，

回答：

□ 一屏是否只有一个核心思想？

□ 是否不存在 Dashboard？

□ 是否不存在 Feature Grid？

□ 是否具有明确方向？

□ 是否保持上下文？

□ 是否保留充足留白？

□ Sticky 是否帮助理解？

□ 是否控制阅读节奏？

□ 是否符合 Scientific Narrative？

□ 用户是否知道下一步应该思考什么？

如果不能全部满足，

页面继续修改。

---

# 本章总结

Layout，

不是空间设计。

Layout，

是认知设计。

真正优秀的页面，

不会告诉用户：

内容在哪里。

而会自然引导用户：

下一步应该思考什么。

BioassayCraft 的 Layout，

不是为了组织页面。

而是为了组织科学推理。
