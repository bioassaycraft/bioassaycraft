Journeys

Structured bioassay practice workflows

Journeys will translate pharmacopoeial requirements into real project paths for method development, validation, and data analysis.

A Journey is not a tool list. It is a guided sequence of decisions, assumptions, interpretation, and next steps that can call Learn artifacts and Tools when needed.

Planned first journeys

From rules to project decisions

  • Method Validation Journey: ATP, design, sample size, acceptance criteria, observed data, and final decision.
  • Method Development Journey: ATP, method principle, critical factors, first experiment, model selection, and readiness for validation.
  • Data Analysis Journey: model selection, variance decomposition, similarity assessment, potency calculation, and reporting judgement.
